Google OAuth 2.0 & Session Restoration

Tài liệu kỹ thuật giải thích cơ chế Đăng nhập/Đăng ký nhanh qua Google kết hợp tính năng tự động phục hồi phiên làm việc bảo mật cao bằng Double Cookie trên MetaFruit.

1. Tổng Quan Giải Pháp

Giải pháp được thiết kế nhằm mang lại trải nghiệm người dùng liền mạch (Seamless UX) nhưng vẫn đảm bảo tính an toàn hệ thống ở mức doanh nghiệp. Người dùng có thể đăng ký tài khoản mới và đăng nhập chỉ với 1 click thông qua cổng bảo mật của Google.

Java 25 Native Jakarta Servlet 6 JWT HMAC-SHA256 HttpOnly Cookie

Trải Nghiệm "Không Gián Đoạn"

Thông thường, các hệ thống sử dụng Session để lưu thông tin đăng nhập. Khi đóng trình duyệt, Session bị huỷ và người dùng phải đăng nhập lại. Bằng cách thiết lập Double Cookie (Access & Refresh Token) và bộ lọc thông minh, MetaFruit sẽ tự động nhận diện và khôi phục trạng thái đăng nhập khi người dùng mở lại trang chủ, tối ưu tỷ lệ mua hàng.

2. Sơ Đồ Luồng Xử Lý (OAuth Flow)

Luồng đăng nhập trao đổi mã token diễn ra hoàn toàn khép kín thông qua 7 bước chuẩn OAuth 2.0:

1

Người dùng click "Đăng nhập nhanh bằng Google"

Được chuyển hướng từ website tới Google OAuth Consent Screen qua GoogleLoginServlet.

2

Google trả về authorization_code

Sau khi người dùng đồng ý cấp quyền, Google redirect về URL Callback /GoogleCallback kèm mã code xác thực.

3

Trao đổi mã code lấy Access Token Google

GoogleCallbackServlet gửi một POST request HTTPS cực nhanh tới cổng Token của Google để lấy access_token.

4

Truy vấn thông tin hồ sơ (User Info)

Sử dụng access_token để gọi Google API lấy email và họ tên đầy đủ của người dùng.

5

Đồng bộ hoá tài khoản (Local DB)

Tìm kiếm tài khoản theo email trong DB. Nếu chưa có, tự động tạo tài khoản với vai trò mặc định CUSTOMER và kích hoạt trạng thái ACTIVE.

6

Thiết lập cookies bảo mật & Cấp phát Session

Hủy session cũ (phòng vệ session fixation), tạo session mới, cấp bộ đôi cookies tự ký mã hóa HMAC-SHA256 (Access Token 15 phút, Refresh Token 7 ngày).

7

Chuyển hướng thông minh

Điều hướng người dùng về trang Dashboard tương ứng với vai trò của họ (CUSTOMER về Home, ADMIN về Admin Panel, v.v.).

3. Phân Tích Code Chi Tiết

A. Xây dựng HTTP Client không dùng thư viện ngoài

Sử dụng cổng java.net.http.HttpClient nguyên bản trong Java 25 giúp gọi các API bên ngoài của Google với hiệu suất tối ưu và hạn chế lỗ hổng từ thư viện bên thứ ba:

HttpClient client = HttpClient.newHttpClient();
HttpRequest request = HttpRequest.newBuilder()
        .uri(URI.create(AppConfig.GOOGLE_LINK_GET_USER_INFO + accessToken))
        .GET()
        .build();
HttpResponse<String> response = client.send(request, HttpResponse.BodyHandlers.ofString());

B. Tự động phục hồi phiên làm việc thông minh

Triển khai SessionRestoreFilter đánh chặn trên /* để tự động phục hồi phiên đăng nhập khi phát hiện cookies, tăng tốc tải trang bằng cách bỏ qua các file tĩnh:

String uri = req.getRequestURI();
// Tối ưu hoá: Bỏ qua kiểm tra cookies với tài nguyên tĩnh
if (uri.contains("/assets/") || uri.endsWith(".css") || uri.endsWith(".js") ...) {
    chain.doFilter(request, response);
    return;
}

4. Các Biện Pháp Bảo Mật Triển Khai

Ứng dụng tuân thủ nghiêm ngặt chuẩn an toàn OWASP Top 10:

Cookies HttpOnly Chống Session Fixation Mã hoá HMAC-SHA256 SQL Parameterized

1. Cookies HttpOnly: Cookies Access & Refresh Token được cấu hình setHttpOnly(true). Điều này ngăn chặn triệt để mã JavaScript của hacker (ví dụ qua lỗi XSS) đọc hoặc ăn trộm token đăng nhập.

2. Phòng thủ Session Fixation: Khi đăng nhập thành công từ Google Callback, Servlet thực hiện session.invalidate() để phá hủy Session ID cũ được sinh ra khi người dùng là Guest, tránh việc hacker cố tình gắn sẵn Session ID để nghe trộm.

3. Ký bảo mật HMAC-SHA256: Access Token được mã hóa dạng userId.expiresAt.signature tự ký bằng thuật toán bảo mật HmacSHA256 với khóa bí mật dài từ Server. Không lưu trực tiếp ID dạng clear-text.
